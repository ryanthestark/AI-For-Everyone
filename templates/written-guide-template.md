## Template Usage Notes

**For Guide Writers:**

1. **Clear objectives** - Readers should know what they'll learn
2. **Step-by-step** - Break down into manageable pieces
3. **Examples** - Show, don't just tell
4. **Screenshots** - Visual aids are helpful
5. **Checkpoints** - Help readers verify progress
6. **Troubleshooting** - Address common issues
7. **Practice exercise** - Reinforce learning
8. **Next steps** - Guide continued learning

**Writing Tips:**

* **Use simple language** - Avoid unnecessary jargon
* **Be specific** - "Click the blue button" not "click the button"
* **Test instructions** - Have someone follow them
* **Include warnings** - Prevent common mistakes
* **Explain why** - Not just how, but why it matters
* **Keep sections focused** - One concept per section

**Formatting:**

* Use headers for clear structure
* Use numbered lists for sequential steps
* Use bullet points for non-sequential items
* Use code blocks for commands/prompts
* Use blockquotes for important notes
* Use emoji sparingly for visual breaks

**Accessibility:**

* Alt text for all images
* Descriptive link text (not "click here")
* Clear structure with headers
* Sufficient contrast in images
* Simple language
* Logical reading order
